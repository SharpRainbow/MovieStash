 Проект представляет собой мобильное приложение-сервис для просмотра информации о кино. Его выполнение разделено на две части: работа с БД и создание графического интерфейса.

## База данных
Работа с базой данных происходит в СУБД PostgreSQL. Для упрощения работы с базой данных используется среда JetBrains DataGrip. Для удобства работы база данных была выгружена на облачный хостинг Render.

База данных была создана в соответствии с предварительно спроектированной ER-диаграммой.
![Alt text](https://i.imgur.com/mQouzgB.png)

В базе данных были созданы функции для добавления, редактирования и удаления записей во все таблицы, функции бана и разбана пользователей, а также для регистрации новых пользователей. Пользователи при этом создаются с уже зашифрованными паролями. Шифрование осуществляется с помощью функции digest(), предоставляемой расширением pg_crypto. Для шифрования используется алгоритм sha256.

Пользователи могут оценивать фильмы по десятибалльной шкале, при этом при добавлении, изменении или удалении оценки на фильм рейтинг этого фильма пересчитывается. Для этого были написаны два триггера – срабатывающий при вставке оценки и срабатывающий при ее изменении или удалении. 

В базе данных также реализованы представления – созданы представления для недавних новостей (за последние 7 дней), недавних фильмов (вышедших за последние 30 дней), а также лучших фильмов по версии пользователей (фильмы с оценкой минимум 8.0).

Для заполнения базы данных была взята информация о фильмах и актерах с сайта-сервиса Кинопоиск. Для этого использовалось неофициальное API Кинопоиска. Был написан парсер, который отправляет запросы к API и сохраняет результаты в формат JSON, который впоследствии был преобразован в данные для вставки в таблицы.

Поскольку формат bytea для изображений серьезно нагружает базу данных, было принято решение хранить картинки для новостей на интернет-хостинге Imgur. На нем достаточно мягкие условия для выгрузки изображений (не более 50 в час, не более 1250 в день). Таким образом, картинка для новости выгружается на хостинг, а в базу данных сохраняется ссылка на эту картинку, которая используется для отображения в приложении. 

Фотографии знаменитостей так же хранятся в виде ссылок, полученных при парсинге через API Кинопоиска. 

Механизмы работы с описанными API можно рассмотреть подробно в классах ImgurResponse, NetworkAPI и WebClient.

## Графический интерфейс

Графический интерфейс для работы с базой данных было решено реализовать в форме мобильного приложения для Android. Для создания приложения используется язык Kotlin в рамках среды Android Studio. Для упрощения создания интерфейса используется система Gradle. Связь с базой данных осуществляется с помощью Java Database Connector (JDBC). Скриншоты работы с приложением представлены в Приложении B.

В приложении была реализована функция регистрации пользователей, функция входа в систему по логину, возможность просмотра информации о фильмах, рецензий, новостей и коллекций, функции создания, редактирования и удаления рецензий, оценок, коллекций и новостей (для модераторов), функция изменения информации об аккаунте, функция бана-разбана пользователей и так далее.

В приложении была реализована защита от SQL-инъекций путем использования подготовленных запросов. Подготовленный (параметризованный) запрос исключает вероятность SQL-атаки за счет предварительной компиляции кода запроса отдельно от данных.

Для работы с изображениями в приложении используется API хостинга изображений Imgur. Приложение позволяет выгружать изображения на хостинг под анонимным пользователем. Ссылки на выгруженные картинки сохраняются и используются в дальнейшем для отображения.

Приложение также фиксирует оценки на фильмы с агрегатора кинокритиков IMDB и с сайта Кинопоиск.
